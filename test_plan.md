1. Issue: Share image generation generates images with black backgrounds and incorrect layout/styling when the application is in dark mode or due to `html-to-image` rendering bugs with Tailwind's dark mode classes.
2. Root Cause: `ShareModal.tsx` uses a specific DOM element `ref={previewRef}` for the image capture. This element has classes like `bg-black`, `border-white/20`, etc. The issue is likely that when `html-to-image` captures the element, it doesn't correctly capture the specific visual styles we want for the shareable image (which should probably have a consistent, high-contrast, beautiful appearance regardless of the user's current theme). Also, `cacheBust: true` and `pixelRatio: 2` are used, but we might need to inline styles or force light/dark mode for the capture.
3. Fix Plan:
   - Update `ShareModal.tsx` to have a dedicated, hidden-from-viewport but rendered-in-DOM capture element, OR explicitly force the styling of `previewRef` to be independent of the current theme (e.g., explicitly dark but with guaranteed visibility).
   - The screenshot showed the title text overflowing or being cut off, and the "AI Portal" / "Read More" being cramped.
   - Adjust the `ShareModal.tsx` capture div to be a fixed size (e.g., 1080x1080 or 1200x630 for standard OG image sizes) and scale it down for preview using CSS transform, so the generated image is high resolution and properly laid out.
